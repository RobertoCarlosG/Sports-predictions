TABLA: teams
------------
  Equipos MLB (identificador = team id de la API oficial).

  id                 INTEGER NOT NULL  PK   id de equipo en statsapi.mlb.com
  name               VARCHAR(128) NOT NULL  nombre completo
  abbreviation       VARCHAR(8) NOT NULL    abreviatura (ej. NYY)
  venue_id           INTEGER NULL           id de estadio en MLB (si aplica)
  venue_name         VARCHAR(256) NULL      nombre del estadio (referencia)

  Índices:
    - PRIMARY KEY (id)
    - idx_teams_venue_id ON (venue_id) WHERE venue_id IS NOT NULL  -- 005


TABLA: games
------------
  Partidos; clave primaria = game_pk de MLB.

  game_pk            INTEGER NOT NULL  PK   id de partido en MLB
  season             VARCHAR(8) NOT NULL    temporada (ej. 2024)
  game_date          DATE NOT NULL          fecha local del partido
  game_datetime_utc  TIMESTAMPTZ NULL       inicio en UTC (si viene de la API)
  status             VARCHAR(64) NOT NULL   estado (Scheduled, Final, Game Over, ...)
  home_team_id       INTEGER NOT NULL       FK → teams.id (local)
  away_team_id       INTEGER NOT NULL       FK → teams.id (visitante)
  venue_id           INTEGER NULL
  venue_name         VARCHAR(256) NULL
  home_score         INTEGER NULL           -- 002_game_scores
  away_score         INTEGER NULL           -- 002_game_scores
  lineups_json       JSONB NULL             feed en vivo / alineaciones (payload)
  boxscore_json      JSONB NULL             boxscore completo (payload)
  home_starter_id    INTEGER NULL           -- 003: probable o boxscore pitchers[0]
  away_starter_id    INTEGER NULL           -- 003


TABLA: game_weather
-------------------
  Condiciones en el estadio por partido (una fila por game_pk).

  id                 SERIAL  PK
  game_pk            INTEGER NOT NULL  UNIQUE FK → games.game_pk (ON DELETE CASCADE)
  temperature_c      DOUBLE PRECISION NULL
  humidity_pct       DOUBLE PRECISION NULL
  wind_speed_mps     DOUBLE PRECISION NULL
  pressure_mbar      DOUBLE PRECISION NULL
  elevation_m        DOUBLE PRECISION NULL
  raw_json           JSONB NULL              respuesta agregada (Open-Meteo, etc.)
  fetched_at         TIMESTAMPTZ NOT NULL    momento de la última actualización


TABLA: game_feature_snapshots
-----------------------------
  Features para entrenamiento / inferencia ML (una fila por partido cuando exista).

  id                   SERIAL  PK
  game_pk              INTEGER NOT NULL  UNIQUE FK → games.game_pk (ON DELETE CASCADE)
  home_wins_roll       DOUBLE PRECISION NULL    rolling win rate (ventana N=10)
  away_wins_roll       DOUBLE PRECISION NULL
  home_runs_avg_roll   DOUBLE PRECISION NULL    runs anotadas avg (ventana N=10)
  away_runs_avg_roll   DOUBLE PRECISION NULL
  temperature_c        DOUBLE PRECISION NULL    snapshot del clima al construir la fila
  humidity_pct         DOUBLE PRECISION NULL
  wind_speed_mps       DOUBLE PRECISION NULL
  elevation_m          DOUBLE PRECISION NULL
  home_win             INTEGER NULL             etiqueta: 1 local, 0 visitante (solo en finales)
  total_runs           DOUBLE PRECISION NULL    etiqueta de regresión (solo en finales)
  home_starter_era     DOUBLE PRECISION NULL    -- 003: ERA de temporada (abridor)
  away_starter_era     DOUBLE PRECISION NULL    -- 003
  home_bullpen_era     DOUBLE PRECISION NULL    -- 003: ERA staff completo (proxy)
  away_bullpen_era     DOUBLE PRECISION NULL    -- 003
  feature_vector_json  TEXT NULL                serialización opcional del vector


TABLA: pitching_era_cache
-------------------------
  Caché de ERA (jugador o equipo) por temporada; evita reconsultar la API en rebuilds.

  id            SERIAL  PK
  kind          VARCHAR(1) NOT NULL CHECK (kind IN ('P','T'))  P=people, T=team
  ref_id        INTEGER NOT NULL                                people_id o team_id
  season        VARCHAR(8) NOT NULL
  era           DOUBLE PRECISION NOT NULL
  updated_at    TIMESTAMPTZ NOT NULL DEFAULT NOW()

  Índices:
    - UNIQUE (kind, ref_id, season)


TABLA: prediction_results
-------------------------
  Caché de inferencia (una fila por partido). GET /predict puede leer aquí si
  model_version coincide con el modelo activo en el servidor.

  game_pk               INTEGER NOT NULL  PK, FK → games.game_pk (ON DELETE CASCADE)
  home_win_probability  DOUBLE PRECISION NOT NULL
  total_runs_estimate   DOUBLE PRECISION NOT NULL
  over_under_line       DOUBLE PRECISION NOT NULL
  model_version         VARCHAR(64) NOT NULL    p. ej. "rf-db-v1@a8b3f2"
  trigger_reason        VARCHAR(64) NULL        motivo del último cálculo
  computed_at           TIMESTAMPTZ NOT NULL DEFAULT NOW()
  -- 004_prediction_evaluation:
  predicted_winner      VARCHAR(10) NULL        "home" | "away"
  actual_winner         VARCHAR(10) NULL        "home" | "away" | "tie"
  is_correct            BOOLEAN NULL            TRUE acierto, FALSE fallo, NULL pendiente
  evaluated_at          TIMESTAMPTZ NULL        cuándo se evaluó

  Índices:
    - idx_prediction_results_computed_at
    - idx_prediction_results_model_version
    - idx_prediction_results_is_correct ON (is_correct) WHERE is_correct IS NOT NULL
    - idx_prediction_results_evaluated_at ON (evaluated_at) WHERE evaluated_at IS NOT NULL


TABLA: model_versions
---------------------
  Historial y estado del modelo de predicción activo.
  Una fila por carga (lifespan al arrancar, /admin/model/reload).

  id                    SERIAL  PK
  model_version         VARCHAR(80) NOT NULL    p. ej. "rf-db-v1@a8b3f2c4"
  base_version          VARCHAR(64) NOT NULL    "rf-db-v1" | "rf-synthetic-v0"
  loaded_at             TIMESTAMPTZ NOT NULL DEFAULT NOW()
  file_mtime            TIMESTAMPTZ NULL
  file_size_bytes       BIGINT NULL
  is_synthetic          BOOLEAN NOT NULL DEFAULT FALSE
  trained_on_games      INTEGER NULL
  val_accuracy_home     DOUBLE PRECISION NULL
  val_mae_total_runs    DOUBLE PRECISION NULL
  val_proba_home_std    DOUBLE PRECISION NULL
  split_mode            VARCHAR(40) NULL
  val_from_requested    DATE NULL
  feature_names_json    TEXT NULL              JSON con FEATURE_NAMES del bundle
  loaded_by             VARCHAR(64) NULL       admin que disparó reload, NULL si auto
  notes                 TEXT NULL
  is_active             BOOLEAN NOT NULL DEFAULT FALSE

  Índices:
    - idx_model_versions_loaded_at  ON (loaded_at DESC)
    - ux_model_versions_active      UNIQUE ON (is_active) WHERE is_active = TRUE
    - idx_model_versions_base_version


TABLA: admin_users
------------------
  Operadores con acceso al panel «Operaciones» (API /api/v1/admin/* y frontend /operations).

  id             SERIAL  PK
  username       VARCHAR(64) NOT NULL UNIQUE
  password_hash  VARCHAR(256) NOT NULL    hash bcrypt
  is_active      BOOLEAN NOT NULL DEFAULT TRUE
  created_at     TIMESTAMPTZ NOT NULL DEFAULT NOW()

  Sin datos semilla en el .sql: alta vía CLI create_admin o POST /api/v1/admin/auth/bootstrap
  (una vez, con ADMIN_BOOTSTRAP_SECRET). Ver backend/sql/README.md.


TABLA: app_users
----------------
  Usuarios finales (Google OAuth) para control de apuestas. Distintos de admin_users.

  id             UUID PK DEFAULT gen_random_uuid()
  google_id      VARCHAR(128) NOT NULL UNIQUE
  email          VARCHAR(256) NOT NULL UNIQUE
  display_name   VARCHAR(256) NULL
  picture_url    TEXT NULL
  is_active      BOOLEAN NOT NULL DEFAULT TRUE
  created_at     TIMESTAMPTZ NOT NULL DEFAULT NOW()
  last_login_at  TIMESTAMPTZ NULL


TABLA: bet_banks
----------------
  Bancos / bankroll por usuario.

  id              SERIAL PK
  user_id         UUID NOT NULL FK → app_users.id (CASCADE)
  name            VARCHAR(128) NOT NULL
  initial_amount  DOUBLE PRECISION NOT NULL
  currency        VARCHAR(8) NOT NULL DEFAULT 'USD'
  is_active       BOOLEAN NOT NULL DEFAULT TRUE
  created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()


TABLA: bet_periods
------------------
  Corte mensual por banco (saldo apertura / cierre).

  id                 SERIAL PK
  bank_id            INTEGER NOT NULL FK → bet_banks.id (CASCADE)
  user_id            UUID NOT NULL FK → app_users.id (CASCADE)
  name               VARCHAR(64) NOT NULL
  year               INTEGER NOT NULL
  month              INTEGER NOT NULL (1–12)
  starting_balance   DOUBLE PRECISION NOT NULL
  closing_balance      DOUBLE PRECISION NULL
  status               VARCHAR(16) NOT NULL DEFAULT 'open'  -- open|closed
  closed_at            TIMESTAMPTZ NULL
  created_at           TIMESTAMPTZ NOT NULL DEFAULT NOW()
  UNIQUE (bank_id, year, month)


TABLA: bets
-----------
  Apuestas registradas (MoneyLine u Over/Under).

  id                 SERIAL PK
  user_id            UUID NOT NULL FK → app_users.id (CASCADE)
  bank_id            INTEGER NOT NULL FK → bet_banks.id (CASCADE)
  period_id          INTEGER NOT NULL FK → bet_periods.id (RESTRICT)
  game_pk            INTEGER NOT NULL FK → games.game_pk (RESTRICT)
  bet_type           VARCHAR(16) CHECK moneyline|over_under
  bet_side           VARCHAR(16) NOT NULL
  stake              DOUBLE PRECISION NOT NULL
  odds               DOUBLE PRECISION NOT NULL
  ou_line            DOUBLE PRECISION NULL
  status             VARCHAR(16) NOT NULL DEFAULT pending
  result_source      VARCHAR(16) NULL   -- local|mlb_api|manual
  result_checked_at  TIMESTAMPTZ NULL
  notes              TEXT NULL
  created_at         TIMESTAMPTZ NOT NULL DEFAULT NOW()


RELACIONES (resumen)
--------------------
  games.home_team_id            → teams.id
  games.away_team_id            → teams.id
  game_weather.game_pk          → games.game_pk    (UNIQUE, CASCADE)
  game_feature_snapshots.game_pk → games.game_pk   (UNIQUE, CASCADE)
  prediction_results.game_pk    → games.game_pk    (PK, CASCADE)
  bet_banks.user_id             → app_users.id
  bet_periods.bank_id           → bet_banks.id
  bet_periods.user_id           → app_users.id
  bets.user_id                  → app_users.id
  bets.bank_id                  → bet_banks.id
  bets.period_id                → bet_periods.id
  bets.game_pk                  → games.game_pk
